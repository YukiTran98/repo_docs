Mỗi workflow được map trực tiếp lên các layer trong Architecture — màu của từng bước tương ứng với layer nào đang xử lý.

Layer 1 · Entry
User query vào Runner
Runner.run(product_agent, "nokia 3210") — asyncio chạy agent với query đầu vào.
Layer 1 · Entry
LLM quyết định gọi tool
OpenAI model nhận query + instructions, tự quyết định gọi rag() tool để tìm thông tin sản phẩm.
Layer 2 · Core Tool
Embed query
get_embedding(query) gọi OpenAI text-embedding-3-small, trả về dense vector. Sau đó normalize bằng L2 norm.
Layer 3 · Infrastructure
Vector search trong ChromaDB
collection.query() tìm top 10 vectors gần nhất trong products collection, trả về metadata.
Layer 2 · Core Tool
Format kết quả
Loop qua metadatas, lấy trường information từ mỗi kết quả, ghép thành chuỗi trả về cho LLM.
Layer 1 · Entry
LLM tổng hợp câu trả lời
LLM nhận kết quả RAG, sinh ra câu trả lời tự nhiên. Runner trả về RunResult object.
Layer 4 · Index Builder
Load product data
Đọc dữ liệu sản phẩm từ CSV/JSON/DB — chưa implement trong build_vector_search.py.
Layer 2 · Core Tool
Embed từng sản phẩm
Gọi get_embedding() cho mỗi sản phẩm, convert text thành dense vector.
Layer 3 · Infrastructure
Upsert vào ChromaDB
Lưu embeddings + metadata (trường information) vào products collection. Sẵn sàng cho query.

04 Onboarding quick start

Install & run

Step 1 — create & activate virtualenv
python -m venv .venv
# Windows:
.venv\Scripts\activate
# macOS / Linux:
source .venv/bin/activate
Step 2 — install dependencies
pip install -r requirements.txt
# chromadb, numpy, pandas cần cài thêm thủ công
pip install chromadb numpy pandas requests
Step 3 — setup env
# tạo file .env
OPENAI_API_KEY=sk-your-key-here
Step 4 — build index & run agent
# implement build_vector_search.py trước, rồi:
python build_vector_search.py
python rag_agent.py
# query mặc định: "nokia 3210"

Key things to know

ChromaDB tại ./db/. PersistentClient lưu local. Collection phải tên là products và phải được populate trước khi chạy agent.

build_vector_search.py hiện còn trống. Đây là phần cần implement: load product data → embed → upsert vào ChromaDB.

Embedding model là text-embedding-3-small. Vector được L2-normalize trước khi query — đảm bảo cosine similarity chính xác.

Framework là openai-agents (Swarm-style). Agent tự quyết định gọi rag() tool dựa trên instructions, không cần hardcode logic.

Metadata schema: information field. Khi upsert vào ChromaDB, mỗi document phải có metadata={"information": "...product text..."}.

Top-10 results. RAG trả về 10 kết quả gần nhất. LLM tổng hợp từ toàn bộ context này để trả lời.
